# 深度学习-优化方法

# 1.梯度下降算法

| 梯度下降算法                            | 定义                      | 缺点                | 优点                                               |
| --------------------------------- | ----------------------- | ----------------- | ------------------------------------------------ |
| BGD(随机梯度下降)                       | 每次迭代都要计算每个样本的损失函数的梯度并求和 | 计算量大,迭代数据慢        | 全局最优化                                            |
| SGD(随机梯度下降)                       | 每次迭代只采集一个样本,计算并且更新参数    | 准确率下降,存在噪点,非全局最优解 | 训练数据块,支持在线学习(<font color="red">实际使用最多的算法</font>) |
| MBGD(小批量梯度下降)</br>(全名 Mini-Batch) | 每次迭代选取一小部分训练样本计算并且更新参数  | 准确率不如BGD,非全局最优解   | 计算小批量数据的梯度更加高效,支持在线学习                            |

例子:

    import tensorflow as tf
    # 实例化优化方法 SGD    learning_rate学习率
    opt=tf.keras.optimizers.SGD(learning_rate=0.1)
    #定义要调整的参数
    var =tf.Variable(1.0)
    # 定义损失函数: 无参但有返回值
    loss=lambda: (var ** 2)/2.0
    # 计算梯度,并对参数进行更新
    opt.minimize(loss,[var]).numpy()
    #参数更新结果
    var.numpy()

# 2.模型训练有三个基础概念

这是一个针对数据集数量的 `调整参数`

| 名词            | 定义                                               |
| ------------- | ------------------------------------------------ |
| Epoch(纪元)     | 使用训练集的全部数据对模型进行一次完整训练                            |
| Batch         | 使用训练集中的艺校部分样本对模型权重进行一次反向传播的参数更新,这一小部分样本被称为`一批数据` |
| Iteration(迭代) | 使用一个Batch数据对模型进行一次参数更新的过程                        |



# 3.反向传播算法(BP算法)

该方法与梯度下降算法相结合,对网络中所有权重计算损失函数的梯度,并利用梯度值来更新权值以最小化损失函数

#### 3.1 前向传播与后向传播

*   前向传播: 指的是数据输入的神经网络中,逐层向前传递,一直到运算到输出层为止
*   反向传播: 从输出层获取的损失函数`Error`,从后往前,依次求各个参数(权重,偏置)的推导,这就是反向传播

## 4.梯度下降优化方法

#### 4.1 动量算法 (Momentum)

*   1.动量梯度下降算法 (<font color="red">对梯度进行修正</font>)
    计算梯度的指数加权平均数,并利用该值来更新参数值.动量梯度下降法的整个过程为,其中β通常设置为0.9
    `与原始的梯度下降算法相比,它的下降趋势更平滑`

keras中设置,momentum参数

    import tensorflow as tf
    # 实例化优化方法 SGD    learning_rate学习率   动态梯度下降momentum
    opt=tf.keras.optimizers.SGD(learning_rate=0.1,momentum=0.9)
    #定义要调整的参数,初始值
    var =tf.Variable(1.0)
    val0=var.value()
    # 定义损失函数: 无参但有返回值
    loss=lambda: (var ** 2)/2.0
    # 一次更新 :计算梯度,并对参数进行更新
    opt.minimize(loss,[var]).numpy()
    val1=var.value()
    # 二次更新: 计算梯度,并对参数进行更新,因为加入了momentum,步长会增加
    opt.minimize(loss,[var]).numpy()
    val2=var.value()

    # 打印两次更新的步长
    print("第一次步长={}".format((val0-val1).numpy()))
    print("第二次步长={}".format((val1-val2).numpy()))

另外还有一种动量算法(NAG)在momentum的基础上进一步加快收敛,提高响应性, 要设置`nesterov`设置为true

*   2.AdaGrad算法 (<font color="red">对学习率进行修正 学习率下降 才是优解</font>)
    AdaGrad就是这样一个“智能调速器”。它会观察每个参数（可以理解为影响你下山路径的每一个因素）的学习情况，如果某个参数在过去的学习中被频繁更新（比如经常遇到陡坡），那么AdaGrad就会减小对这个参数的更新步长，让模型的学习更加稳定；反之，如果某个参数很少被更新（比如一直在平地走），AdaGrad就会增大对这个参数的更新步长，加快学习的速度。

`AdaGrad通过动态调整学习率来帮助模型更高效地学习`

    import tensorflow as tf
    # 实例化优化方法 Adagrad  
    opt=tf.keras.optimizers.Adagrad(learning_rate=0.1,initial_accumulator_value=0.1,epsilon=1e-06)
    #定义要调整的参数,初始值
    var =tf.Variable(1.0)
    # 定义损失函数: 无参但有返回值
    loss=lambda: (var ** 2)/2.0
    # 进行更新
    opt.minimize(loss,[var]).numpy()
    #参数更新结果
    var.numpy()

## 5.学习率退火

在训练神经网络时,一般情况下学习率都会随着训练而变化,这主要是由于,在神经网络训练的后期,如果学习率过高,会造成loss的震荡,但是如果学习率减少的过快,又快造成收敛变慢的情况.

#### 5.1 分段常数衰退 `PiecewiseConstantDecay`

分段常数衰退是在事先定义好的训练次数区间上,设置不同的学习率常数.刚开始学习率大一些,之后越来越小,区间的设置需要配根据样本量调整,一般样本量越大区间间隔应该越小
对应方法:

    tf.keras.optimizers.schedules.PiecewiseConstantDecay(boundaries,values)

    参数:
    boundaries:设置分段更新的step值
    values: 针对不用分段的学习率值

例子: 对于前10000步,学习率为1.0,接下来的10000-110000步,学习率为0.5,之后学习率为0.1

    import tensorflow as tf

    # 设置的分段的step值
    boundaries=[100000,110000]
    # 不同的step对应的学习率
    values=[1.0,0.5,0.1]
    # 实例化进行学习的更新
    learing_rate_fn=tf.keras.optimizers.schedules.PiecewiseConstantDecay(boundaries,values)
