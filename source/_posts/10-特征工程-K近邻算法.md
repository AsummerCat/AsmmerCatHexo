# K近邻算法

    使用 scikit-learn库进行应用

## 1.算法介绍

KNN算法  (一般来说是欧式距离)
```
*   定义:
    一个样本在特征空间中k个最相似(也就是特征空间中最邻近)的鱼坛币中的大多数属于某一个类别
    主要用于(距离定位)

场景: 假设 我不知道自己属于哪个区,A处于海淀区,B处于仓山区, 我距离A比较近 我也就属于海淀区
```
<!--more-->

## 2.scikit-learn实现K近邻算法
*   API
    sklearn.neighbors.KNeighborsClassifier(n\_neighbors=5,algorithm='auto')



    参数: n_neighbors  int默认等于5 , 邻居数=5
    algorithm : 
         默认为auto 让算法自己决定使用的搜索方法 
         brute:蛮力搜索,也就是线性扫描,当训练集很大的时候,计算非常耗时
         kd_tree:构建kd树存储数据以便对其进行快速检索的树形数据结构
         ball_tree: 克服kd树高纬失效的

from sklearn.neighbors import KNeighborsClassifier

# 构造数据集
x= [[1],[10],[20],[40],[50]]
y= [0,0,1,1,2]

# 机器学习模型训练
# 实例化API->实例化一个估计器对象
estimator =KNeighborsClassifier(n_neighbors=1)
# 使用fit方法进行训练
## x:特征值 y:目标值
estimator.fit(x,y)

## 模型评估
#方法1:对比真实值和预测值
ret=estimator.predict([[4]])
print(ret)
## 返回的结果[0] 接近[1]


## 3.距离度量

#### 3.1常见距离公式

###### 3.1.1 欧式距离(Euclidean Distance)

一般来说这个就是 直接算->空间中 两点之间的距离

###### 3.1.2 曼哈顿距离(Manhattan Distance)

也称为城区距离 (一个十字路口开到另外一个十字路口)的驾驶距离  (有障碍)

###### 3.1.3 切比雪夫距离(Chebyshev Distance)

国际象棋中-> 一个棋子可以走8个方向

###### 3.1.4 闵可夫斯基距离(Minkowski Distance)

这个不是距离 是一种距离的定义 ,根据P值不一样 可以换位不同的参数
p=1 曼哈顿距离
p=2 欧式距离
p=无穷大  切比雪夫距离

###### 3.1.5 存在明显缺点

闵可夫斯基距离,欧式距离,曼哈顿距离,切比雪夫距离 都存在明显的缺点
不考虑 各个分量的分布(期望,方差)

###### 如果存在`连续属性`和`离散属性`的距离计算
```
若属性之间存在有序关系 ,则可以转换为连续值
比如: `身高属性: "高","中等","矮"`,可转换为{1,0.5,0}
闵可夫斯基距离可以用于有序属性

若属性值之间不存在有序关系,则通常将其转换为向量的方式
比如:`性别属性: 男,女`可转换为 {{1,0},{0,1}}
```
## 4.K值的选择
```
KNN算法中 K值的大小选择对模型的影响

*   K值过小:
    容易收到异常点的影响
    容易过拟合

*   K值过大:
    收到样本均衡的问题
    容易欠拟合
```

## KNN算法总结

```
*   优点:
    1.简单有效
    2.重新训练的代价低 (不需要输出保存模型)
    3.适合类域交叉样本
    4.适合大样本自动分类

*   缺点:
    1.懒惰学习 ,KNN算法是懒惰学习方法,一些积极学习的算法要快很多
    2.类别评分不是规格化
    3.输出可解释性不强  ,例如决策树的输出\`可解释性就较强
    4.对不均匀的样本不擅长
    5.计算量较大

```
