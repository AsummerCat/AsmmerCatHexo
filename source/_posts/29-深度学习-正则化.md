# 深度学习-正则化

不仅要求在训练集上误差小,而且希望在新样本上的泛化能力强
许多机器学习算法都采用相关的策略来减少测试误差,这些测试略被统称为正则化.
`正则化通过对算法的修改来减少泛化误差`

`L1与L2正则化` ,`Droupout正则化`,`提前停止`,`批标准化`

<!--more-->


## 1.L1与L2正则化

是最常见的正则化方法.他们在损失函数中增加一个正则项.一定程度上减少了过拟合

*   L1正则化 ->使不太重要的权重为0
    `tf.leras.regularizers.L1(l1=0.01)`

*   L2正则化 ->使权重的值变小趋近于0
    `tf.leras.regularizers.L2(l2=0.01)`

*   L1L2正则化
    `tf.leras.regularizers.L1L2(l1=0.01,l2=0.01)`

我们可以直接在某一层的layers中指明正则化类型和超参数即可
`kernel_regularizer=regularizers.l2(0.001)`

    import tensorflow as tf
    from tensorflow.keras import regularizers

    # 定义模型的输入
    inputs= tf.keras.Input(shape=(3,),name="input")
    # 第一层: 激活函数为relu,其他为默认
    x=tf.keras.layers.Dense(10,kernel_regularizer=regularizers.l2(0.001),  activation="relu",name="layer1")(inputs)
    # 第二层: 激活函数为relu,其他为默认
    x=tf.keras.layers.Dense(2, kernel_regularizer=regularizers.l1(0.001), activation="relu",name="layer2")(x)

    x=tf.keras.layers.Dense(2, kernel_regularizer=regularizers.L1L2(0.001,0.001), activation="relu",name="layer3")(x)
    # 第三层:(输出层) 激活函数为Sigmoid
    outputs=tf.keras.layers.Dense(2, activation="sigmoid",name="layer4")(x)
    # 使用Model来创建模型,指明输入和输出
    model=tf.keras.Model(inputs=inputs,outputs=outputs,name="根据Model创建模型")

    # 展示模型结果
    model.summary()

## 2.Droupout正则化 (最常用的正则化技术)

原理: 在每个迭代过程中,随机选择某些节点,并且删除前向和后向连接

*   使用:

<!---->

    tf.keras.layers.Dropout(rate)

    参数:
     rate: 每一个神经元被丢弃的概率

例子:

    import tensorflow as tf
    import numpy as np
    # 定义dropout层,每一个神经元都有0.2的概率被失活,未被失活的输入将按照1/(1-rate)放大
    layer=tf.keras.layers.Dropout(0.2,input_shape=(2,0))
    # 定义五个频次的数据
    data=np.arange(1,11).reshape(5,2).astype(np.float32)
    #原始数据打印
    print(data)
    # 进行随机失活: 在training模式下,返回应用dropout后的输出;或者在非training模式下,正常返回输出
    outputs=layer(data,training=True)
    # 打印失活后的结果
    print(outputs)

## 3.提前停止

是将一部分训练集作为验证集.当验证集的性能越来越差时或者性能不再提升.则立即停止对该模型的训练,这叫做提前停止

在`tf.keras`中,我们可以使用`callbacks`函数实现早期停止

```
import tensorflow as tf

tf.keras.callbacks.EarlyStopping(monitor='val_loss',patience=5)
# monitor表示检测量 val_loss表示验证集损失,
# patience表示epochs数量

```

案例:

```
import tensorflow as tf
import numpy as np
# 当连续3个epoch loss不下降则停止训练
callback=tf.keras.callbacks.EarlyStopping(monitor='val_loss',patience=5)
# 定义只有一层的神经网络
model=tf.keras.models.Sequential([tf.keras.layers.Dense(10)])
# 设置损失函数和梯度下降算法
model.compile(tf.keras.optimizers.SGD(),loss='mse')
# 模型训练
history = model.fit(
    np.arange(100).reshape(5, 20),  # 输入数据
    np.array([0, 1, 2, 1, 2]),      # 目标数据
    epochs=10,                      # 最大训练轮数
    batch_size=1,                   # 批量大小
    callbacks=[callback],           # 回调列表
    verbose=1                       # 输出训练过程
)

# 打印实际运行的 epoch 数量
print("实际运行的 epoch 数量:", len(history.history['loss']))

```

## 4. 批标准化

批标准化（Batch Normalization，简称BN）是一种在深度学习中广泛使用的技巧，用于加速训练过程并提高模型的稳定性。它通过规范化每一层的输入，使得模型在训练过程中更加稳定，减少了内部协变量偏移（internal covariate shift）的问题。

使用:

        # BN层直接放入构建升级脑网络的结构中即可
    tf.keras.layers.BatchNormalization(epsilon=0.001,center=True,scale=True,beta_initializer='zeros',gamma_initializer='ones',name='layer2')

    import numpy as np
