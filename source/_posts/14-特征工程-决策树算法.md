# 决策树

决策树 可以当做规则引擎理解

## 1.决策树的实现过程

决策树: 是一种树形结构,其中每个内部节点表示一个属性上的判断,每个分支代表一个判断结果的输出,最后每一个叶节点表示一种分类结果,本质是一颗由多个判断节点组成的树

对这个过程进行量化,需要用到信息论中的 `信息熵` `信息增益`

*   信息熵 是用来度量数据集不确定性的指标，信息熵越大表示不确定性越高。
*   信息增益 是用来评估特征对数据集纯度提升程度的指标，信息增益越大表示该特征对分类的贡献越大。
    在决策树算法中，通过计算信息增益来选择最佳的分割特征，从而构建高效的决策树模型。

## 2.决策树分类原理

#### 2.1 熵

*   概念:  表示`混乱`程度的量度

*   系统越有序或者集中,熵值越低; 系统越混乱或者分散,熵值越高

*   信息熵

*   是用来度量数据集不确定性的指标，信息熵越大表示不确定性越高。

<!--more-->


#### 2.2 信息增益

决策树算法
应用场景
`ID3 算法`：最早的决策树算法之一，使用信息增益作为特征选择的标准。
`C4.5 算法`：改进版的决策树算法，使用信息增益率（Gain Ratio）来避免偏向于选择具有较多取值的特征。
`CART 算法`：使用基尼不纯度（Gini Impurity）而非信息熵，但在某些变体中也会使用信息增益。
随机森林（Random Forest）：虽然随机森林使用的是多个决策树的集成，但在单个决策树的构建过程中，仍然可以使用信息增益作为特征选择的方法。

*   信息增益: ` (ID3 算法)`
    是以某特征划分数据集前后的差值来计算的
    比如: 信息增益表示得知特征X的信息,而使得类Y的信息熵减少的程序
    (`也就是说 会优先选择分类多的数据`)

*   信息增益率 `(C4.5 算法)`
    比如: 信息增益率准则对可取值数目较多的属性有所偏好,为了避免这种偏好可能带来的影响 使用`增益率`来选择最优划分属性
    `增益率越大,表示纯度提升越大,该数据越重要`
    采用`后剪枝方法`
    ps: C4.5 算法只适合于能够驻留于内存的数据集,当训练集大得无法在内存容纳时,程序无法运行
    ps: 构建树的过程中,需要对数据集进行多次的顺序扫描和排序,因而导致算法低效

*   基尼不纯度 (`CART 算法`)  ->分类和回归任务都可用
    `基尼值Gini(D)`表示:从数据集D中随机抽取两个样本,其类别标记不一致的概率,`故,Gini(D)值越小,数据集D的纯度越高`
    `基尼值Gini_index(D)`: 一般,选择使划分后基尼系数最小的属性作为最优化分属性
    案例:
    基尼值就是一种测量你有多接近这个目标的方法。如果你随便挑两个苹果，发现它们的颜色不一样，那么说明你的苹果混合得很厉害，基尼值就高。反之，如果你几乎每次都能挑到同样颜色的苹果，说明你的苹果分得不错，基尼值就低。

## 3.cart剪枝

剪枝是决策树学习算法对付`过拟合`的主要手段

*   剪枝原因

<!---->

    * 噪音,样本冲突,即错误的样本数据
    * 特征即属性不能完全作为分类标准
    * 巧合的规律性,数据量不够大

分为: `预剪枝`  `后剪枝`

`主动去掉一些分支来降低过拟合的风险`

*   `预剪枝` :
    指在决策树生成过程中,对每个节点在划分前先进行估计,若当前节点的划分不能带给决策树`泛化`性能提升,则停止划分并将当前节点标记为叶节点

<!---->

    在构建树的过程中,同时剪枝
    * 限制节点最小样本数
    * 指定数据高度
    * 指定熵值的最小值

\*`后剪枝`:
指`先从训练集生成一颗完整的决策树,然后自底向上的对非叶节点进行考察`,若建该节点对应的子树替换为叶节点能带来决策树`泛化`性能提升,则将该子树替换为叶节点

对比来说:

*   `后剪枝`通常比`预剪枝`决策树保留了更多的分支
*   一般情况下,`后剪枝`决策树的欠拟合风险很小,`泛化`性能往往优于与`预剪枝`决策树
*   但是`后剪枝`训练过程是在生成决策树之后的,并且自底向上的,所以训练时间开销比`未剪枝`决策树和`预剪枝`决策树都要大得多

## 4.决策树算法API

    class sklearn.tree.DecisionTreeClassifier(criterion='gini',max_depth=None,random_state=None)

    参数: 
    *criterion : 特征选择标准  'gini' 或者'entropy' 前者代表基尼系数,后者代表信息增益  默认gini

    *min_samples_split: 内部节点再划分所需要最小样本数 
    默认是2  ,如果样本数量级非常大,则推荐增大这个值 
    (参考案例) 10万样本 ,选择min_samples_split=10

    *min_samples_leaf: 叶子节点最少样本数
    默认是1  ,如果样本数量级非常大,则推荐增大这个值 
    (参考案例) 10万样本 ,min_samples_leaf=5

    *max_depth : 决策树最大深度  
    如果模型样本量多,特征也多的情况下,推荐限制这个最大深度,具体取值取决于数据的分布.常用的可以取值10-100之间

    *random_state: 随机数种子 保证每次结果一致

#### 4.1 案例 决策树分类器
```
`from sklearn.tree import DecisionTreeClassifier`

    # 决策数算法
    from sklearn.tree import DecisionTreeClassifier
    from sklearn.datasets import load_iris
    from sklearn.model_selection import train_test_split
    from sklearn.metrics import accuracy_score

    # 加载数据集
    data = load_iris()
    X = data.data
    y = data.target

    # 划分训练集和测试集
    X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

    # 实例化决策树分类器
    clf = DecisionTreeClassifier(random_state=42)

    # 训练模型
    clf.fit(X_train, y_train)

    # 预测
    y_pred = clf.predict(X_test)

    # 计算准确率
    accuracy = accuracy_score(y_test, y_pred)
    print("Accuracy:", accuracy)
```

## 5.图形化决策树 export\_graphviz

`export_graphviz` 是 scikit-learn 库中用于导出决策树图形的一种方法。它允许你将决策树模型导出为 Graphviz 文件格式，从而可以在 Graphviz 工具中可视化决策树的结构。

#### 5.1 安装依赖 graphviz

    pip install scikit-learn graphviz

    conda install scikit-learn graphviz

#### 5.2 安装开源软件包

    # Ubuntu/Debian
    sudo apt-get install graphviz

    # macOS (使用 Homebrew)
    brew install graphviz

    # Windows (使用 Chocolatey)
    choco install graphviz

#### 5.3 相关API
```
直接使用这个的话 需要把输出的结果放到三方网站上去查看图像化结果
`from sklearn.tree import DecisionTreeClassifier, export_graphviz`

    # 导出决策树为 Graphviz 文件(.dot)
    dot_data = export_graphviz(

    clf ,这里表示实例化的分类器
    out_file=None：不保存到文件，而是返回字符串。  如果导出文件的化就是'/1/xxx.dot`

    class_names：类别名称。
    filled=True：节点填充颜色。
    rounded=True：节点边角圆润。
    special_characters=True：支持特殊字符。



    # 使用 Graphviz 渲染决策树
    import graphviz 
    graph = graphviz.Source(dot_data)
    graph

    # 使用 Graphviz 渲染并保存为 PDF 文件
```
