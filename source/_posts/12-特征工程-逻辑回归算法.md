# 1.逻辑回归 Logistic Regression  是分类模型

\*\* 线性回归: 主要用于预测
\*\* 逻辑回归主要用于分类

逻辑回归(Logistic Regression) 是机器学习中的`一种分类模型`,逻辑回归是一种分类算法

#### 1.1 逻辑回归应用场景

*   广告点击率
*   是否为垃圾邮件
*   虚假账户
*   是否的问题

都是属于两个类别之间的判断,逻辑回归就是解决二分类问题的利器
<!--more-->

#### 1.2 逻辑回归原理

逻辑回归的输入就是`线性回归的的结果`

*   判断标准
    \*\* 回归的结果输入到`sigmoid`函数中
    \*\* 输出结果: \[0,1]区间中的一个概率值,默认为0.5为阀值



#### 1.3 损失及其优化

*   在逻辑回归中,当预测结果不对的时候,如何衡量其损失\\

*   损失
    逻辑回归的损失,称之为`对数似然损失`
    无论何时,我们都希望`损失函数值,越小越好`

*   优化
    同样使用梯度下降算法,去减少损失函数的值 ,这样去更新逻辑回归前面对应算法的权重参数

提升属于1类别概率,降低原本是0类别的概率

## 2.逻辑回归API

```
sklearn.linear_model.LogisticRegression(solver='liblinear',penalty='l2',C=1.0)
参数:
 * solver:  可选{'liblinear','sag','saga','newton-cg','lifgs'}
    1.默认: 'liblinear' 用于优化问题的算法
    2.对于小数据集来说'liblinear'是个不错的选择,而`sag`和`saga`对于大型数据会更快
    3.对于多类问题 ,只有'sag','saga','newton-cg','lifgs'可以处理多项损失, (多分类问题)
      `liblinear`仅限于`one-versus-rest`分类(单分类)

 * penalty: 正则化的种类  l2 岭回归之类的梯度下降算法
    可选的值包括 'l1'、'l2'、'elasticnet' 和 'none'。
        'l1' 表示 L1 正则化，倾向于产生稀疏解，即很多权重变为零。
        'l2' 表示 L2 正则化，倾向于产生平滑解，即权重较小但不为零。
        'elasticnet' 是 L1 和 L2 的组合，同时具备两者的特点。
        'none' 表示不使用正则化。
    
 * C :正则化力度

```

ps: 默认将类别数量少的当做正例

*   `LogisticRegression`方法相当于`SGDClassifier(loss="log",penalty=""`,
*   `SGDClassifier` 实现了一个普通的随机梯度下降学习,
*   使用`LogisticRegression`这是(实现了SAG)(随机平均梯度下降算法)

## 3.分类评估方法

`准确率`和`召回率`

#### 3.1 混淆矩阵

在分类任务中,预测结果与正确标记之间存在四种不同的组合,构成混淆矩阵(适用于多分类)

列为`真实结果`
行为`预测结果`

|    | 正例    | 假例    |
| -- | ----- | ----- |
| 正例 | 真正例TP | 伪反例FN |
| 反例 | 伪正例FP | 真反例TN |

真正例TP
真反例TN

#### 3.2 精确率和召回率

*   准确率(对不对): 公式: (TP+TN)/(TP+FN+FP+TN)

*   精确率(查的准不准): 预测结果为正例样本中真实为正例的比例
    公式: (TP)/(TP+FP)

|                             | <font color="red">正例</font>    | 假例    |
| --------------------------- | ------------------------------ | ----- |
| <font color="red">正例</font> | <font color="red">真正例TP</font> | 伪反例FN |
| 反例                          | <font color="red">伪正例FP</font> | 真反例TN |

*   召回率(查的全不全): 真实为正例的样本中预测结果为正例的的比例
    公式: (TP)/(TP+FN)

|                             | <font color="red">正例</font>    | 假例                             |
| --------------------------- | ------------------------------ | ------------------------------ |
| <font color="red">正例</font> | <font color="red">真正例TP</font> | <font color="red">伪反例FN</font> |
| 反例                          | 伪正例FP                          | 真反例TN                          |

#### 3.3 F1-score

反映了模型的稳健性

    F1= 2TP/(2TP+FN+FP)

#### 3.4分类评估报告API

    参数:
       * y_true: 真实目标值

       * babels: 指定类别对应的数字
       * target_names: 目标类别名称

## 4.ROC曲线的绘制-ROC曲线与AUC指标

`样本不均衡的情况下,会导致不管怎么预测都是正例`
数据在 1:4的情况下就可以当做是样本不均衡了

#### 4.1 TPR与FPR

*   TPR=TP/(TP+FN)   (召回率)
    所有真实类别为1的样本中,预测类型为1的比例

*   FPR=FP/(FP+TN) (预测错误的情况)
    所有真实类别为0的样本中,预测类型为1的比例

#### 4.2 ROC曲线

*   ROC曲线的横轴就是`预测错误的情况 FPR` ,纵轴就是`预测正确的情况 召回率 TPR`
*   两者相等时,表示的意义则是: 对于不论真实类别是1还是0的样本,分类器预测为1的概率是相等的,此时`AUC`为0.5  ->也就是无论是什么样的数据 都预测为一个固定值

#### 4.3 AUC指标

*   AUC的概率意义是随机取一堆正负样本,正样本得分大于负样本得分的概率
*   AUC的范围在\[0,1]之间,并且越接近1越好,越接近0.5属于乱猜
*   AUC=1,完美分类器,绝大部分场景下不存在
*   0.5\<AUC<1,由于随机猜测. 这个分类器(模型)妥善设定阀值的话, 能有预测价值
*

#### 4.4 AUC计算API roc\_auc\_score

```
from sklearn.metrics import roc_auc_score
 
 roc_auc_score(y_true,y_score)
 *计算ROC曲线面积,也就是AUC值
 参数: 
 y_true: 每个样本的真实类别,必须为0(反例),1(正例)标记
 y_score: 预测得分,可以是正类的估计概率,置信值,或者分类器方法的返回值
 
 
```

## 5.输出指标 classification\_report

`from sklearn.metrics import classification_report`

```
from sklearn.datasets import make_classification
from sklearn.model_selection import train_test_split
from sklearn.metrics import classification_report
from sklearn.ensemble import RandomForestClassifier
from imblearn.over_sampling import RandomOverSampler
import matplotlib.pyplot as plt
from collections import Counter

# 生成不平衡数据集
X, y = make_classification(
    n_samples=5000,           # 样本总数
    n_features=2,             # 特征数量
    n_informative=2,          # 信息量丰富的特征个数
    n_redundant=0,            # 冗余特征个数
    n_repeated=0,             # 重复特征个数
    n_classes=3,              # 类别数量
    n_clusters_per_class=1,   # 每个类别中的簇数
    weights=[0.01, 0.05, 0.94], # 类别权重
    random_state=0            # 随机种子
)

# 查看原始数据集的类别分布
print("原始数据集类别分布:")
print(Counter(y))

# 数据可视化
plt.scatter(X[:, 0], X[:, 1], c=y)
plt.title("原始数据集")
plt.show()

# 划分训练集和测试集
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

# 使用 RandomOverSampler 进行过采样
ros = RandomOverSampler(random_state=42)
X_resampled, y_resampled = ros.fit_resample(X_train, y_train)

# 查看过采样后的类别分布
print("过采样后的数据集类别分布:")
print(Counter(y_resampled))

# 数据可视化
plt.scatter(X_resampled[:, 0], X_resampled[:, 1], c=y_resampled)
plt.title("过采样后的数据集")
plt.show()

# 创建随机森林分类器
rf_clf = RandomForestClassifier(n_estimators=100, random_state=42)

# 训练模型
rf_clf.fit(X_resampled, y_resampled)

# 预测测试集
y_pred = rf_clf.predict(X_test)

# 输出分类报告
print(classification_report(y_test, y_pred))
```
